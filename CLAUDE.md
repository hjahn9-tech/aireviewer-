# CREX AI Reviewer — 프로젝트 규칙

## 기본 원칙

- 모든 산출물 및 서비스의 기본 언어는 **한국어**로 제공한다.

- 작업은 항상 **단계적**으로 진행하며, 이전 단계의 Exit Criteria를 충족한 후 다음 단계로 이행한다.

- 작업 완료 후 **즉시 배포하지 않는다**.

- 코드 변경 시 항상 **TC(테스트 케이스)를 먼저 작성**하고, 코드 레벨 테스트 수행 결과를 리포팅한다.

- 사용자가 직접 검증해야 하는 항목은 명시적으로 안내한다.

- 환경변수는 반드시 `.env` 파일로 관리하며, `.gitignore`에 포함한다.

- 개인정보/보안 키는 소스코드에 하드코딩하지 않는다.

## 기술 스택

- Frontend: React + Vite + TypeScript

- DB: Supabase (MCP 연동)

- 배포: Vercel

- 소스코드: GitHub

## 작업 기록

- 모든 작업 완료 후 `project_log.md`를 업데이트한다.

- 각 항목: 날짜, 작업 내용, 변경 파일, 테스트 결과, 다음 단계

## 진행 단계

1. 프로젝트 계획 및 초기화
2. Agent 및 Skill 정의
3. DB 연동 (Supabase)
4. 로컬 테스트
5. GitHub 연동 및 Vercel 배포
